Capabilities · Complete Guide

Core Engine · What This Platform Actually Is ▼
Boardwalk Playground is not a dashboard. It is a **local-first software factory**. It can generate entire apps, Workers, APIs, UIs, templates, and full project structures.

This is the machine that builds the apps you want: music generator, comic studio, rideshare, laundry, skating, Roll Show, video generator — all of it.

You define the blueprint. The Playground builds the project, files, Worker, API calls, UI pages, and deploy pipeline.
Project Workspace Engine ▼
This engine manages multiple full-stack apps at once. Each app lives in its own folder under `/projects`.

**How you use it:** - Create a folder → it becomes a project - Switch projects instantly - Everything (edit, run, deploy) happens inside the selected project
This is how you run a whole ecosystem from one dashboard.
File System Engine ▼
This engine can create, edit, delete, and generate files and folders. It is the backbone of all app generation.

**It can generate:** - HTML, CSS, JS - Workers - API routes - Config files - Templates - Entire project structures
This is how the Playground builds apps from scratch.
Worker Generator & Cloudflare Engine ▼
The Playground can generate Cloudflare Workers, connect them, and deploy them.

**It can create:** - worker.js - wrangler.toml - API routes - Bindings (D1, R2, KV, Queues) - Deployment scripts

**It can deploy:** - Workers - Pages - D1 - R2 - KV - Queues
This is how your apps get real backends.
API Engine · Call ANY API ▼
The Playground can generate API clients and connect to ANY external API.

**It can call:** - AI APIs - Music APIs - Video APIs - Payment APIs - Map APIs - Weather APIs - ANY REST or GraphQL endpoint

**It can generate:** - Worker → external API calls - Frontend → Worker → external API calls - Authenticated requests - Streaming responses
This is how your apps talk to the world.
UI Generator · Pages & Panels ▼
The Playground can generate UI pages for any domain.

**It can create:** - Dashboard panels - Full pages - Forms - Editors - Media viewers - Custom interfaces
This is how each app gets its own UI.
