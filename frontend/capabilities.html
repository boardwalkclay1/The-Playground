Capabilities

Project Workspace Engine ▼
This is the heart of the Playground. You can have multiple full-stack projects living inside your environment at the same time. The Project Switcher on the dashboard lets you instantly jump between them.

**How you use it:** - Create a folder inside `/projects` - It instantly appears in the Project Switcher - Select it, and now EVERYTHING you do (edit, deploy, run) happens inside that project
This is how you manage multiple apps like a real studio.
File System Access ▼
The FileTree panel shows you the entire structure of your selected project. Click any file and it opens instantly in the inline editor.

**How you use it:** - Click a file → it opens on the right - Edit it → hit Save - Want deeper editing? Open the full Editor page
This is your local VS Code, but inside your dashboard.
Inline Dashboard Editor ▼
This is for quick edits. You don’t have to leave the dashboard.

**How you use it:** - Click a file in the FileTree - It opens in the main view - Edit → Save
Perfect for fast fixes, config tweaks, and small changes.
Full Editor Page ▼
When you need to focus, this is your zone. Full-screen, distraction-free, deep editing.

**How you use it:** - Go to `/editor` - Type the project name - Type the file path - Load → Edit → Save
This is your “real coding” workspace.
Cloudflare Suite ▼
This is your deployment engine. You can deploy Workers, Pages, D1, R2, KV, and Queues — all from your dashboard.

**How you use it:** - Use the quick panel for fast deploys - Or open the full Cloudflare page for detailed control - Logs appear instantly so you know what happened
This is how you ship apps without leaving the Playground.
GitHub Integration ▼
Bring any GitHub repo straight into your Playground.

**How you use it:** - Enter a repo URL - Choose a project name - Clone
The repo becomes a project you can edit, run, and deploy.
Run / Deploy Engine ▼
This is how you execute your apps locally.

**How you use it:** - Select a project - Click “Run Backend” or “Run Frontend” - Logs appear in the main view
This is your local dev server, but controlled from the dashboard.
Plugin System ▼
Plugins let you expand the Playground into ANYTHING.

**How you use it:** - Add a plugin folder - It appears in the Plugins page - Plugins can add panels, pages, or backend logic
This is how you scale the Playground into a full ecosystem.
Preview Panel ▼
The iframe on the dashboard is your live preview window.

**How you use it:** - Point it at your local app - Point it at generated pages - Use it to test UI instantly
This is your built-in browser.
Logs & Diagnostics ▼
Every action the Playground takes can be logged.

**How you use it:** - Open `/logs` - See system events, deploy logs, builder logs
This is your debugging center.
Theme Engine ▼
Switch between Industrial and Neon Hacker instantly.

**How you use it:** - Click the Theme button - The entire UI updates instantly
This is your visual identity switch.
