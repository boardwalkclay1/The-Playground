Capabilities · Complete System Manual

Core System Engines

Core Engine · What Boardwalk Playground Is ▼
Boardwalk Playground is a **local-first software factory**. It generates, edits, runs, previews, and deploys entire apps. It is the machine that builds your ecosystem: music generator, comic studio, rideshare, laundry, skating, Roll Show, video generator, and more.
Project Workspace Engine ▼
Manages multiple full-stack apps at once. Each app lives under `/projects`. Switch projects instantly. All editing, running, previewing, and deploying happens inside the selected project.
File System Engine ▼
Creates, edits, deletes, and generates files and folders. Generates HTML, CSS, JS, Workers, APIs, templates, and full project structures.
Editor Engine ▼
Full-screen industrial editor. Supports load, save, new file, delete file. Connected directly to the File System Engine.
Preview Engine ▼
Live preview of any project. Auto-refresh. Error capture. Connected to Logs + Errors.
Logs Engine ▼
Streams system logs, generator logs, and error logs in real time. Used by every engine in the Playground.
App Generator Engine ▼
Generates entire apps from prompts. Creates folders, files, Workers, APIs, UI pages, templates, and deploy pipelines. Logs every step.
Cloudflare Suite ▼
Deploy Workers, Pages, D1, R2, KV, Queues. Inspect bindings. Validate wrangler.toml. Fully integrated with Logs.
Microcontroller Lab · MCU Lab Pro ▼
Full breadboard simulator. Board awareness (ESP32, Arduino, RP2040). Component awareness (LEDs, sensors, motors). Wire drawing, auto-routing, pin validation. Code generation from wiring. Firmware flashing.

Word Glossary · Definitions

Panel ▼
A full-screen cinematic section of the Playground UI. Panels replace pages.
Project ▼
A folder under `/projects` containing a full app: frontend, backend, assets, config.
Worker ▼
A Cloudflare backend script that handles APIs, logic, storage, and deployment.
Binding ▼
A connection between a Worker and a Cloudflare resource (D1, R2, KV, Queue).
D1 ▼
Cloudflare’s SQLite database. Used for structured data.
R2 ▼
Cloudflare’s object storage. Used for media, files, exports, uploads.
KV ▼
Cloudflare’s key-value store. Used for config, flags, small data.
Queue ▼
Cloudflare’s job queue system. Used for background tasks.
Generator ▼
The engine that creates apps, files, Workers, and UI from prompts.
MCU ▼
Microcontroller Unit. Used in the MCU Lab for hardware simulation.
Pin Map ▼
A list of all pins on a microcontroller board and what they support.
Auto-Routing ▼
Automatic wire drawing between components in the MCU Lab.
Preview ▼
Live rendering of your app inside the Playground.
Logs ▼
Real-time output from the system, generator, Worker, and errors.
Bubble Field ▼
Boardwalk’s signature rounded input style used across the UI.
Industrial Theme ▼
The default Boardwalk theme: dark, neon, metallic, cinematic.
